# Sun-Readable Light Theme Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reconfigure Ghostty, Neovim, Starship, and Lualine so light mode uses high-contrast Solarized Light with no transparency (sun-readable), dark mode uses TokyoNight Night uniformly, and macOS appearance changes automatically swap palettes and per-mode opacity in already-open Ghostty windows.

**Architecture:** Ghostty's base config is light-mode-safe (full opacity); a launchd agent reacts to `AppleInterfaceThemeChangedNotification` and runs `sync-terminal-theme`, which writes/removes `~/.config/ghostty/overrides` to layer dark-mode transparency back in, writes `~/.config/starship/active-palette` for fish to read on shell start, and sends Ghostty `SIGUSR2` to hot-reload its config. Neovim picks colorscheme + transparency at startup via an existing `os_is_dark()` shell-out.

**Tech Stack:** Ghostty (terminal), Neovim/LazyVim (editor), Starship (prompt), fish (shell), launchd (macOS service), bash (sync script).

## Pre-flight notes for the implementing engineer

- This dotfiles repo uses symlink-style stowing. `~/.config/ghostty`, `~/.config/nvim`, `~/.config/fish`, and `~/.config/starship.toml` are already symlinks into the repo, so editing files in the repo takes effect immediately — no `install.sh` re-run needed for config changes.
- `~/bin` is a symlink to `<repo>/bin/bin/`. Anything dropped there is on `$PATH`.
- The Ghostty theme directory does **not** include a plain "Solarized Light" — the canonical built-in is **`iTerm2 Solarized Light`** (verified via `ghostty +list-themes`). Use that name exactly.
- The dark theme name is **`TokyoNight Night`** (with capitals and the space — already used in the current config).
- Ghostty reloads its config on receiving `SIGUSR2`. If a future Ghostty version changes this, fall back to invoking the `reload_config` keybind via `osascript`. We assume `SIGUSR2` works (confirmed in current stable).
- Repo working directory for all commands: `/Users/johnchristopher/gitprojects/dotfiles`.

## Task 4: Write the `sync-terminal-theme` script

**Files:**
- Create: `bin/bin/sync-terminal-theme`

- [ ] **Step 1: Write the script**

Create `bin/bin/sync-terminal-theme` with exactly this content:

```bash
#!/usr/bin/env bash
# sync-terminal-theme — invoked by launchd when macOS appearance changes,
# also runs at login. Idempotent: safe to run repeatedly.
#
# Behavior:
#   Dark mode  → copy overrides.dark to overrides, write tokyonight palette, signal Ghostty
#   Light mode → remove overrides, write solarized_light palette, signal Ghostty

set -euo pipefail

GHOSTTY_DIR="$HOME/.config/ghostty"
OVERRIDES_SRC="$GHOSTTY_DIR/overrides.dark"
OVERRIDES_DST="$GHOSTTY_DIR/overrides"
STARSHIP_DIR="$HOME/.config/starship"
PALETTE_FILE="$STARSHIP_DIR/active-palette"

mkdir -p "$STARSHIP_DIR"

# `defaults read` exits non-zero when AppleInterfaceStyle is unset (= Light).
if defaults read -g AppleInterfaceStyle >/dev/null 2>&1; then
    appearance="dark"
else
    appearance="light"
fi

case "$appearance" in
    dark)
        cp "$OVERRIDES_SRC" "$OVERRIDES_DST"
        printf 'tokyonight\n' > "$PALETTE_FILE"
        ;;
    light)
        rm -f "$OVERRIDES_DST"
        printf 'solarized_light\n' > "$PALETTE_FILE"
        ;;
esac

# Reload all running Ghostty processes (no-op if Ghostty isn't running).
pkill -SIGUSR2 -x ghostty 2>/dev/null || true

# Write a tiny breadcrumb so launchd-driven runs are debuggable.
LOG_DIR="$HOME/.local/state/sync-terminal-theme"
mkdir -p "$LOG_DIR"
printf '%s\t%s\n' "$(date '+%Y-%m-%d %H:%M:%S')" "$appearance" >> "$LOG_DIR/log"
```

## Task 6: Add `terminal-theme-sync` install action to `install.sh`

- [ ] **Step 1: Add the install function**

Insert this function in `install.sh`, immediately before the final `case "$1" in` block:

```bash
setup_terminal_theme_sync() {
    title "Installing terminal-theme-sync LaunchAgent"

    local template="$DOTFILES/bin/com.jc.terminal-theme-sync.plist.template"
    local target="$HOME/Library/LaunchAgents/com.jc.terminal-theme-sync.plist"

    if [ ! -f "$template" ]; then
        error "Template not found at $template"
    fi

    mkdir -p "$HOME/Library/LaunchAgents"
    mkdir -p "$HOME/.local/state/sync-terminal-theme"

    sed "s|__HOME__|$HOME|g" "$template" > "$target"
    info "Wrote $target"

    # Replace any prior agent of the same Label, then load fresh.
    launchctl bootout "gui/$(id -u)" "$target" 2>/dev/null || true
    launchctl bootstrap "gui/$(id -u)" "$target"
    launchctl enable "gui/$(id -u)/com.jc.terminal-theme-sync"

    success "terminal-theme-sync agent loaded"

    info "Running sync once to set initial state"
    "$HOME/bin/sync-terminal-theme" || warning "Initial sync failed — agent will still trigger on appearance change"
}
```

# Set Starship palette from active-palette breadcrumb (written by sync-terminal-theme).

function _set_starship_palette

Replace the entire contents of `nvim/.config/nvim/lua/plugins/catppuccin.lua` with:

```lua
-- Picks the active LazyVim colorscheme based on macOS appearance:
--   Dark  → tokyonight-night     (matches Ghostty dark theme)
--   Light → solarized-osaka-day  (high-contrast Solarized Light variant)
--
-- Catppuccin is no longer the active scheme but is still useful as a soft
-- fallback if other schemes fail to load — keeping the spec but without
-- transparent_background so it doesn't fight light-mode contrast.
local os_is_dark = function()
  return (vim.call(
    "system",
    [[echo $(defaults read -globalDomain AppleInterfaceStyle &> /dev/null && echo 'dark' || echo 'light')]]
  )):find("dark") ~= nil
end

return {
  {
    "catppuccin/nvim",
    name = "catppuccin",
    opts = {
      transparent_background = false,
      integrations = {
        alpha = true,
        cmp = true,
        gitsigns = true,
        illuminate = true,
        indent_blankline = { enabled = true },
        lsp_trouble = true,
        mini = true,
        native_lsp = {
          enabled = true,
          underlines = {
            errors = { "undercurl" },
            hints = { "undercurl" },
            warnings = { "undercurl" },
            information = { "undercurl" },
          },
        },
        navic = { enabled = true },
        neotest = true,
        noice = true,
        notify = true,
        neotree = true,
        semantic_tokens = true,
        telescope = true,
        treesitter = true,
        which_key = true,
        dap = { enabled = true, enable_ui = true },
      },
    },
  },
  {
    "LazyVim/LazyVim",
    opts = function(_, opts)
      if os_is_dark() then
        opts.colorscheme = "tokyonight-night"
      else
        opts.colorscheme = "solarized-osaka-day"
      end
    end,
  },
  {
    "nvim-lualine/lualine.nvim",
    event = "VeryLazy",
    opts = {
      options = { theme = "auto" },
    },
  },
  {
    "rcarriga/nvim-notify",
    opts = {
      background_colour = "#1a1b26",
    },
  },
}
```
